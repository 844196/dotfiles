# Handover in a new tmux pane

現在の会話をハンドオーバー文書にまとめ、一時ファイルに書き出し、tmux の新しいペインでまっさらな Claude Code セッションを起動して引き継ぐ。

## 手順

### 1. 会話内容をハンドオーバー文書にまとめる

文書のフォーマットは `/handover` スキルの「ハンドオーバー文書の作成」セクションと同一。整理の観点も同じ:

- 何を解決・実現しようとしていたか
- どこまで進んだか（完了した作業、変更したファイル）
- 調査で判明した事実
- 未解決の問題や残タスク
- ユーザーが示した判断や好み

### 2. 一時ファイルに書き出す

パスは `"${TMPDIR:-/tmp}/handover-$(openssl rand -hex 8).md"` のように組み立てる。ファイル本体は Write ツールで新規作成する（存在しないパスへの Write は Read → Write の往復が不要）。

`.agents/handovers/` は使わない（別セッションで消化された後は不要な一時ファイルなので、リポジトリに残さない）。

### 3. tmux の新しいペインで新セッションを起動

以下のコマンドを実行する（`<handover-file>` は手順2で生成したパス）:

```bash
claude-yolo --split-pane '@<handover-file> を読んで次のステップから作業を続けて'
```

コマンド実行後、「tmux の新ペインに引き継いだ」とだけ報告する。

## 注意事項

- ハンドオーバー文書は別セッションからの参照を前提とする。文脈を共有しない読者が理解できるよう、省略せずに記述する。
- 会話中にコードの変更を行っていた場合、その変更内容も文書に含める。
