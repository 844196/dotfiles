# 「動作確認済み」と書くための条件

実装・修正の完了報告で「動作確認済み」「動く」「直った」と書くには、以下を全て満たす:

1. 実際にコマンド・テストを実行した
2. 期待する出力・状態を確認した
3. エラーが発生していないことを確認した (終了コード、ログ、画面表示)

## これは動作確認ではない

- 「コードを読んだ限り動くはず」
- 「型は通った」「ビルドは成功した」(コードが正しいことは必ずしも機能として正しいことを意味しない)
- 「ライブラリのドキュメントを見る限り合っているはず」
- 「以前に似た実装で動いた」

これらしか確認していない場合は、その範囲を明示する (e.g. 「型チェックは通ったが実行は未確認」)。

## 確率的に再現するバグの場合

「1 回成功した」を完了とせず、再現条件の安定性まで言及する。「N 回中 M 回成功」のように粒度を明示し、N が小さすぎないかも自覚する。

## 検証手段が無いとき

環境的に実行できない・検証手段がない場合は、その旨をはっきり書く。「動くと思う」を「動いた」と書き換えない。
