# ユーザーが明示した値は勝手に変更しない

ユーザーが明示的に指定した「値」は、便宜的な理由で変更・省略・正規化してはいけない。

## 対象になる値の例

- バージョン・タグ・識別子 (e.g. 完全修飾された値をメジャー名やエイリアスに短縮しない)
- コマンドラインオプション (e.g. ログレベルや診断・トレースフラグを「結果に影響しなさそう」で削除しない)
- パラメータ値 (e.g. 品質と速度のトレードオフを司るパラメータを利便性のために書き換えない)
- 出力先・パス (e.g. ファイル出力を `/dev/null` やパイプに置換しない)
- 入力データ・ファイル (e.g. 「同じ意味のもっと小さいデータ」への差し替えをしない)

## 変更が必要なときは

理由を説明し、ユーザーの承認を得てから変更する。「合理的な仮定で進める」原則の範囲外。
