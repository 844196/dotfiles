# ユーザーが明示していないスコープ判断が必要になったら作業を中断する

ユーザーから受けた明示的な指示を達成する過程で、その指示には明示されていないが「スコープに関する判断」(依頼範囲の縮小・拡大・指示文の解釈変更) が必要だと気づいたら、その時点で作業を中断してユーザーに確認を求める。

この原則は permission mode を問わず常に適用される。

## 対象になるケース

- **縮小**: 「全てのファイルを修正」と指示されたが、一部のファイル群を除外したい理由が出てきた
- **拡大**: 「ファイル A の B を直して」と指示されたが、B を直すには C も触らないと整合しないと判明した
- **解釈変更**: ユーザーの指示文の解釈に複数の妥当な候補があり、どれを選ぶかで成果物が大きく変わる

## やってはいけないこと

- ユーザー確認を取らずに「今回はスコープ外とした」「ここは触らないことにした」と宣言だけして進める
- permission mode を理由に確認を省略する
  - e.g. auto モードの「合理的な仮定で進める」原則を依頼範囲の調整まで広げる。同原則の対象は実装上の細部 (命名、ライブラリ選択、軽微なリファクタ等) であって、ユーザーが明示した依頼範囲そのものの調整ではない
- 保証ゲートに任せて先に進める。スコープ判断はメインエージェント自身の責務であり、保証ゲートはこの責務を肩代わりするものではない

## 確認の取り方

検出した不確かさと暫定判断 (このまま縮小/拡大/特定解釈で進める案) の両方を提示する。ユーザーが採否を即断できるよう、選択肢間の差分 (含まれるファイル、変えるコードの粒度、波及範囲等) を具体的に書く。
