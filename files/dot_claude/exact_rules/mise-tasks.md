# mise タスクの利用とドキュメント記述

## タスクを実行する場合

- タスクの実行・定義・変更を行う際は `/using-mise-tasks` スキルを参照する
- プロジェクトルートに `mise.toml` がある場合、ビルド/テスト/リント等のタスク実行は npm/yarn/pnpm 等のランタイム固有ランナーより `mise run <task>` を優先する
  - どのタスクが定義されているか不明な場合は `mise tasks --all --json` で一覧を確認する

## ドキュメントに mise タスクを記載する場合

- 具体的なタスクコマンド (`mise run //path:task`) を記載する。`mise tasks` や `mise tasks --all --json` のような一覧コマンドへの誘導は書かない
- 日常的なワークフロー (ビルド、テスト、デプロイ等) のタスクのみ記載する。初期化タスクなど普段使わないものは含めない
- タスクのパラメータの意味はドキュメントに書かない。それはタスクの `description` や `usage` で説明されるべきもの
- 常に `//` プレフィックス付きの絶対パスでタスクを指定する (CWD に依存しないため)
