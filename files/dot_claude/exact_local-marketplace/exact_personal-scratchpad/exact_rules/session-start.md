# 個人スクラッチパッドディレクトリ

`<repository-root>/.844196/` は、ユーザー個人の補助的なファイル (調査レポート、プラン文書、ハンドオーバー、スクリプト等) を置くスクラッチパッドディレクトリ。

## 書き出す場合

該当する用途のファイルを書き出すときは `<repository-root>/.844196/` 以下に置く。プロジェクトにまだ無ければ新規に作る。

サブディレクトリは目的別に分けて、直下にバラバラに置かない。既存のサブディレクトリに合致するものがあればそれを使い、無ければ目的を表す新しいサブディレクトリを作る。

よくある用途:
```
<repository-root>/.844196/
  plans/                    # 実装計画
  reports/                  # 調査・分析のレポート
  handovers/                # ハンドオーバー・テイクオーバースキルが読み書きするハンドオーバー文書
  logs/                     # 実験のために書き出したログ
  scripts/                  # ユーザーが個人で使う補助スクリプト
```

## 不可逆操作の前は対象を確認する

このディレクトリは gitignore されているため、誤って上書き・削除しても `git restore` や `git checkout` で復元できない。本体システムプロンプトが警告する破壊的操作の例は Git で追跡されているファイルを前提にしており、ここではその救済策が効かない。

破壊的操作 (上書き、削除、移動など) を行う前に:

- 移動先に同名ファイルが存在しないか確認する
- 上書きの影響範囲を確認する
- 必要なら事前にバックアップを取る

衝突や懸念を見つけたら、独断で進めずユーザーに報告する。
