checks:
  - id: doc-consistency
    when: "ゴール達成のための作業内容に .md ファイルの追加・変更・削除が含まれていた"
    body: |
      編集された Markdown について、以下 4 観点で点検する。複数 Edit を重ねたターンや新規セクション追加時は、編集対象を頭から通読してから判断する。

      - **章立てと階層**: h1/h2/h3 の順序と粒度が論理順に並んでいるか。ツギハギ・重複した見出しが残っていないか
      - **整合性**: 文書内に矛盾・用語の揺れがないか。相互参照先や同ディレクトリ内の関連ファイルとも辻褄が合っているか
      - **明確さと根拠**: 略称や代名詞が後から読み返したときに何を指すか分かるか。「判明」「確定」等の断定にエビデンスが付いているか
      - **指示外の追加をしていないか**: 頼まれていない「結論」「まとめ」セクションを足していないか。比較セクションで対象間の記載項目が揃っているか
  - id: doc-followup
    when: "ゴール達成のための作業内容に非 .md ファイル (実装コード、設定、データ等) の追加・変更・削除が含まれていた"
    body: |
      実装ファイルの変更を踏まえ、エージェント向けドキュメントが古くなっていないか確認する:

      - `CLAUDE.md` (プロジェクト概要、コマンド、ディレクトリ構成)
      - `.claude/rules/` (規約、ルール)
      - `docs/` (アーキテクチャ、設計判断)
  - id: deleted-references
    when: "ゴール達成のための作業内容にファイル削除または rename (旧パスの消失) が含まれていた"
    body: |
      削除済みファイル / 旧パスへの参照・リンクがドキュメント (`CLAUDE.md` / `.claude/rules/` / `docs/`) に残っていないか確認する。
