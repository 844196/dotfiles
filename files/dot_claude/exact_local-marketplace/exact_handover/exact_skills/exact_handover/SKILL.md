# Handover

現在の会話をハンドオーバー文書にまとめ、後で読み返して再開できる形でファイルへ書き出す。

## 手順

### 1. 会話内容を整理する

これまでの会話を以下の5観点で整理する:

1. **最終ゴール**: ユーザーが最終的に達成したいこと。手段ではなく目的。
2. **セッション中の指示・制約**: ユーザーがセッション中に明示的に伝えた作業上のルール、好み、制約。CLAUDE.md や rules に書かれていない、このセッション固有の指示。
3. **経緯**: ここに至るまでに何を試し、何を決め、何が分かったか。完了した作業、変更したファイル、調査で判明した事実を含む。
4. **現在地**: 今この瞬間、何が完了していて何が未完了か。未解決の問題は何か。
5. **次の選択肢**: 次にやり得ることの候補を列挙する。「次にやること」を断定しない。判断はユーザーと新セッションのエージェントに委ねる。

現セッションが `takeover` スキルの呼び出しから始まっている場合は、その前のハンドオーバー文書を Read で読む。**「最終ゴール」と「セッション中の指示・制約」は原則として前のハンドオーバー文書からそのまま引き継ぐ**。新しい情報でゴールや制約が変わった場合のみ更新する。これによりハンドオーバーを N 回チェーンしてもゴールや指示が薄れない。

### 2. 空ファイルを作成する

次のファイルパスに空ファイルを作成する: ${user_config.HANDOVER_OUTPUT_PATH_FORMAT}

- Bash コマンド置換ライクなプレースホルダーは、適宜実行し解決すること
- 中間ディレクトリが存在しない場合は `mkdir -p` で作成すること

### 3. 本文を書く

フォーマット:
~~~md
---
created_at: "$(date --iso-8601=seconds)" # 実行して解決すること
claude_session_id: ${CLAUDE_SESSION_ID} # プリプロセッシング済み。そのまま転記すること。
---

# <タイトル>

## 最終ゴール
...

## セッション中の指示・制約
...

## 経緯
...

## 現在地
...

## 次の選択肢
...

## 参考ファイル・リンク
...
~~~

**注意**:
- 文脈を共有しない読者が理解できるよう、**省略せずに記述する**。
- 「最終ゴール」は手段や中間成果物ではなく、**ユーザーが本当に達成したいことを書く**。チェーンハンドオーバーを重ねてもゴールが薄れないよう、前のハンドオーバー文書から引き継ぐことを優先する。
- 「セッション中の指示・制約」は会話の連続性を保つために重要。ユーザーが「〜しないで」「〜のスタイルで」と伝えた内容を漏らすと、新セッションで同じ指摘を繰り返させることになる。
- 会話中にコードの変更を行っていた場合、その変更内容も「経緯」に含める。

## 最終処理

書き出したハンドオーバ文書のパスをユーザーに伝える。
