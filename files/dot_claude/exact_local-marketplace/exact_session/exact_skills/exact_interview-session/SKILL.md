# セッションインタビュー

終了したセッションに `--resume -p` で質問を投げ、当事者視点の振り返りを得る。

## 使い方

```
/interview-session <session-id> [質問]
```

引数はセッションID（UUID形式）1つと、任意の自然言語の質問。UUID以外のテキストを質問として扱う。

例:
- `/interview-session c3007db3-...` — デフォルトの質問セットで振り返り
- `/interview-session c3007db3-... なんでyamlパーサーをnpmから入れようとした？` — 特定の判断について聞く
- `/interview-session c3007db3-... ツールのI/Fで困ったところある？` — 改善ヒントを引き出す

同じセッションIDに対して複数回呼べる。前回の質問・回答がセッションに蓄積されるので、フォローアップ質問が可能。

## 手順

### 1. 引数のパース

UUID形式のトークンをセッションIDとして抽出し、残りを質問テキストとする。

### 2. 質問の実行

```bash
claude --resume <session-id> -p "<質問>"
```

質問テキストが省略された場合、以下のデフォルト質問を使う:

```
このセッションを振り返って、以下の点を教えてほしい:
1. うまくいかなかった箇所はどこか？何が原因だったか？
2. どういう情報・ツール・ドキュメントがあれば、もっとスムーズにできたか？
3. ユーザーに介入された箇所があれば、なぜ自分では正しい判断ができなかったか？
```

### 3. 回答の表示

`claude --resume -p` の出力をそのまま表示する。

必要に応じて回答を踏まえたフォローアップ質問を提案する。
