# Committing Staged Changes

ステージ済みの変更のみをコミットします。ユーザーへの確認なしに直接コミットを実行することが許可されています。

## 手順

### 1. ステージ済みの変更を確認

`git diff --cached --stat` 実行結果:
!`git diff --cached --stat`

`git diff --cached` 実行結果:
!`git diff --cached`

ステージ済みの変更がない場合は、その旨をユーザーに伝えて終了します。

### 2. コミットメッセージ規約の把握

次の優先度でプロジェクトのドキュメントを確認し、コミットメッセージ規約を把握します:

1. `CLAUDE.md` ファイルの内容
2. `AGENTS.md` ファイルの内容
3. `CONTRIBUTING.md` ファイルの内容

プロジェクトのドキュメントにコミットメッセージ規約が見つからない場合は、直近のコミット履歴を確認して規約を推測します。

```bash
git log --oneline -10
```

### 3. コミットメッセージの生成

差分の内容、コミット履歴から読み取れる規約、そしてユーザーから追加の説明がある場合はそれも考慮し、プロジェクトの規約に沿ったコミットメッセージを生成します。

**ルール:**
- コミットメッセージは !`git commit-language` で生成する
- subject は変更の目的を簡潔に表す
- body は原則不要。変更の意図が subject だけでは伝わらない場合のみ付与する

ユーザーからの追加の説明:
$ARGUMENTS

### 4. コミットの実行

```bash
git commit -m "<title>" [-m "paragraph"] [-m "paragraph"] -m "Co-Authored-By: ..."
```
